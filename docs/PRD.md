# 차량 및 부속실 예약 관리 시스템 MVP PRD

## 핵심 정보

**목적**: 조직 내 차량과 부속실(회의실, 창고 등)의 예약 충돌을 없애고 현황을 한눈에 파악할 수 있는 통합 예약 관리 시스템 제공
**사용자**: 사내 차량과 부속실을 주기적으로 예약해서 사용하는 임직원 및 이를 관리하는 담당자

---

## 사용자 여정

```
1. 랜딩/로그인 페이지
   ↓ 로그인 버튼 클릭

2. 로그인 페이지
   ↓ 인증 성공

   [일반 사용자] → 예약 현황 페이지 → 예약 생성/조회/취소
   [관리자]      → 관리자 대시보드 → 리소스 등록/관리/예약 현황 전체 조회
   ↓

3. 예약 현황 페이지 (캘린더 뷰)
   ↓ 예약하기 버튼 클릭

   [차량 선택] → 차량 예약 생성 페이지 → 예약 완료 → 예약 현황 페이지
   [부속실 선택] → 부속실 예약 생성 페이지 → 예약 완료 → 예약 현황 페이지
   ↓

4. 내 예약 목록 페이지
   ↓ 예약 상세 클릭

5. 예약 상세 페이지
   ↓ 수정/취소 선택

   [수정] → 예약 수정 페이지 → 저장 → 예약 상세 페이지
   [취소] → 취소 확인 모달 → 예약 취소 완료 → 내 예약 목록 페이지
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 예약 현황 조회 (캘린더) | 차량/부속실의 전체 예약 현황을 캘린더 뷰로 표시 | 예약 충돌 방지를 위한 핵심 기능 | 예약 현황 페이지 |
| **F002** | 차량 예약 생성 | 날짜/시간/목적지/목적을 입력하여 차량 예약 | 서비스 핵심 가치 제공 | 차량 예약 생성 페이지 |
| **F003** | 부속실 예약 생성 | 날짜/시간/사용 목적을 입력하여 부속실 예약 | 서비스 핵심 가치 제공 | 부속실 예약 생성 페이지 |
| **F004** | 예약 수정 | 기존 예약의 날짜/시간/사용 목적 변경 | 사용자 실수 수정 지원 | 예약 수정 페이지 |
| **F005** | 예약 취소 | 예약 취소 및 슬롯 반환 | 미사용 리소스 반환을 위한 필수 기능 | 예약 상세 페이지 |
| **F006** | 내 예약 목록 조회 | 본인이 생성한 예약 목록 조회 | 사용자 예약 이력 관리 | 내 예약 목록 페이지 |
| **F007** | 리소스 가용성 확인 | 선택한 날짜/시간에 예약 가능한 차량/부속실 목록 표시 | 예약 충돌 방지 핵심 로직 | 차량 예약 생성 페이지, 부속실 예약 생성 페이지 |

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|

### 3. MVP 이후 기능 (제외)

- 예약 알림 (이메일/SMS/푸시)
- 반복 예약 (매주 정기 예약)
- 차량 반납 확인 및 주행 기록 등록
- 외부 캘린더 연동 (Google Calendar 등)
- 예약 승인 워크플로우 (결재선)
- 리소스 사용 통계 및 리포트
- 모바일 앱 (네이티브)

## 메뉴 구조

```
일반 사용자 메뉴 (로그인 후)
├── 예약 현황 - F001 (캘린더 뷰로 전체 예약 조회)
│   ├── 차량 예약하기 버튼 - F002, F007
│   └── 부속실 예약하기 버튼 - F003, F007
├── 내 예약 - F006 (내 예약 목록 및 상세)
│   ├── 예약 수정 - F004
│   └── 예약 취소 - F005
└── 로그아웃 - F010

관리자 메뉴 (로그인 후)
├── 대시보드 - F014 (전체 예약 현황 및 관리)
├── 차량 관리 - F012 (차량 등록/수정/비활성화)
├── 부속실 관리 - F013 (부속실 등록/수정/비활성화)
├── 사용자 관리 - F011 (사용자 역할 변경)
└── 로그아웃 - F010

비로그인 사용자
├── 로그인 - F010
└── 회원가입 - F010
```

## 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `F010` | **접근:** 비로그인 사용자 전용, 로그인 시 자동 리디렉션

| 항목 | 내용 |
|------|------|
| **역할** | 사용자 인증 전용 페이지 |
| **진입 경로** | 앱 최초 접근 시 자동 리디렉션, 상단 로그인 메뉴 클릭 |
| **사용자 행동** | 이메일/비밀번호 입력 후 로그인, 회원가입 페이지로 이동 |
| **주요 기능** | • 이메일/비밀번호 유효성 검사<br>• JWT 토큰 발급 및 로컬 저장<br>• 역할(Role)에 따른 분기 리디렉션<br>• **로그인** 버튼 |
| **다음 이동** | 일반 사용자 → 예약 현황 페이지, 관리자 → 관리자 대시보드, 실패 → 에러 메시지 표시 |

## MVP 범위 외 항목 (Out of Scope)

| 항목 | 제외 이유 |
|------|----------|
| 예약 알림 (이메일/SMS) | 외부 서비스 연동 복잡도 증가, MVP 이후 추가 |
| 반복 예약 기능 | UI/로직 복잡도 높음, 핵심 가치 외 기능 |
| 차량 반납 확인 및 주행 기록 | 운영 프로세스 확정 후 추가 |
| 예약 승인 워크플로우 | 결재선 설계 필요, MVP 이후 추가 |
| 외부 캘린더 연동 | OAuth 연동 복잡도, MVP 이후 추가 |
| 사용 통계 및 리포트 | 데이터 축적 후 의미 있는 기능 |
| 다국어 지원 | 초기 사용자 그룹이 단일 언어권 |
| 소셜 로그인 | 조직 내부 서비스로 자체 인증으로 충분 |
| 모바일 앱 (네이티브) | 웹 반응형으로 MVP 커버 가능 |

## 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| **응답 속도** | 일반 API 응답 2초 이내, 가용성 조회 1초 이내 |
| **동시성 처리** | 동일 리소스 동일 시간대 중복 예약 방지 (DB 레벨 유니크 제약 또는 비관적 락 적용) |
| **인증 보안** | JWT Access Token 만료 시간 1시간, Refresh Token 7일 |
| **접근 제어** | 관리자 전용 API는 ROLE_ADMIN 검증 필수, 본인 예약 외 수정/취소 차단 |
| **반응형 UI** | 모바일/태블릿/데스크톱 3 브레이크포인트 대응 |
| **에러 처리** | API 에러 응답은 code + message 구조 통일, 프론트엔드 토스트 메시지 표시 |
